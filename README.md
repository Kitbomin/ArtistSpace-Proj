# [프로젝트 기획 및 기술 설계서 v1.2]

**서비스명:** ArtistSpace (아티스트 스페이스)

**부제:** AI 저작권 보호 및 활동 기반 뱃지 시스템을 결합한 창작자 특화 SNS

---

## 1. 프로젝트 개요

### 1.1 배경 및 목적

* **창작자 친화적 생태계 구축:** 기존 SNS의 저작권 취약성과 커미션 플랫폼의 복잡함을 해결하는 통합 플랫폼.
* **활동 데이터의 가치화:** 사용자의 업로드 활동을 분석하여 전문성을 나타내는 뱃지를 자동 부여.
* **안전한 창작물 거래:** 플랫폼 중개형 포인트 시스템(에스크로)을 통한 신뢰 기반 커미션 환경 제공.

### 1.2 핵심 타겟

* 그림, 웹툰, 글, 음악, 공예 등 예체능 분야의 모든 창작자 및 서포터.

---

## 2. 주요 기능 명세 (MVP 핵심 범위)

### 2.1 활동 기반 창작 카테고리 뱃지 시스템

* **정의:** 특정 분야 활동이 많은 사용자의 전문성을 시각화하는 자동 인증 시스템.
* **로직:** 특정 카테고리 게시물 비율 40% 이상 + 최소 10개 이상 업로드 시 뱃지 자동 부여.
* **노출:** 프로필, 게시물 카드, 검색 결과 등에 표시되어 작가 브랜딩 강화.

### 2.2 AI 저작권 보호 및 보안 시스템

* **AI 유사도 검사:** 업로드 시 AI 서버 연동 분석. 85% 경고, 95% 이상 시 업로드 보류 및 관리자 검토.
* **콘텐츠 보호:** 자동 워터마크 삽입, 우클릭/드래그 방지, 모바일 캡처 차단 API 연동.
* **19금 필터링:** 가입 시 생년월일 기준, 미성년자 대상 성인 콘텐츠 차단 및 블러 처리.

### 2.3 커미션 및 안전 결제 (에스크로)

* **포인트 거래:** 충전된 포인트를 사용하여 커미션 거래 진행.
* **안심 결제:** 의뢰 시 포인트 선차감 및 플랫폼 보관 → 작업물 최종 승인 시 작가에게 포인트 지급.
* **진행 관리:** 신청 대기 → 작업 중 → 1차 완료 → 수정/최종 완료 단계별 트래킹.

---

## 3. 기술 스택 및 인증 구조

### 3.1 기술 스택

* **Frontend:** React, Axios, React-Router
* **Backend:** Java, Spring Boot, JPA
* **Database:** MySQL (RDB)
* **File Storage:** AWS S3
* **AI Module:** Python (FastAPI), PyTorch

### 3.2 인증 시스템 (Session-based)

* **방식:** `HttpSession`을 이용한 세션 기반 인증.
* **구현:** 서버 메모리 세션 저장 및 클라이언트 `JSESSIONID` 쿠키 발급.
* **Interceptor:** 로그인 여부를 체크하여 미인증 사용자의 권한 페이지 접근 차단.
* **선정 사유:** 초기 개발 단계에서의 직관적인 상태 관리 및 구현 용이성 고려.

---

## 4. 데이터베이스 설계 (Core Entities)

* **User:** 이메일, 암호화 비밀번호, 닉네임, 생년월일, 뱃지 상태, 보유 포인트 등.
* **Post:** 작성자ID, 카테고리, 제목, 내용(HTML), 미디어URL, 성인물 여부, 유사도 점수.
* **Commission:** 신청자/작가ID, 거래 포인트, 진행 상태, 요청 사항, 리뷰.
* **Interaction:** 좋아요, 북마크(공개/비공개), DM(1:1 채팅 내역).

---

## 5. 단계별 로드맵

1. **Phase 1 (MVP):** 세션 인증 기반 회원가입/로그인, 이미지 업로드, 유사도 체크, 에스크로 로직.
2. **Phase 2 (고도화):** 뱃지 자동 계산 스케줄러, 웹툰 전용 뷰어, 실시간 알림 시스템.
3. **Phase 3 (확장):** 유료 회차 결제, AI 학습 방지 기술 적용, 굿즈 판매 연동.
